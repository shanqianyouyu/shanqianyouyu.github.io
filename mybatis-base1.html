学习mybatis过程中做的一点笔记，一方面有助于学习，一方面方便捡起来。

1.类型别名是为 Java 类型设置一个短的名字。 它只和 XML 配置有关，存在的意义仅在于用来减少类完全限定名的冗余。

例如：(mapper文件中)

<select id="listAll" resultType="hello.User">
    SELECT id,name,salary FROM t_user
</select>

resultType=”hello.User” 如果包名很长，（此处包名是hello，比较短）那么不是写一堆吗，很麻烦就有了别名，mybatis定义在mybatis-config中.

上面的别名是精确到类，如果类很多，那么还是相对麻烦了，就有了指定包名，MyBatis 会在包名下面搜索需要的 Java Bean

<typeAliases>
  <package name="domain.blog"/>
</typeAliases>

每一个在包 domain.blog 中的 Java Bean，在没有注解的情况下，会使用 Bean 的首字母小写的非限定类名来作为它的别名。如果还是想在此基础上，将名字改成自己像定义的名字，比如 domain.blog.Author 的别名为 author；若有注解，则别名为其注解值。例子如下：

@Alias("author")
public class Author {

}

一般为了方便看，所以是不会起自己的名字，最好就是类名，易于辨别，避免使用此注解。

2.结果映射

如果忘记写 resultType=”int”抛出以下异常

 Cause: org.apache.ibatis.executor.ExecutorException: A query was run and no Result Maps were found for the Mapped Statement 'hello.UserMapper.queryForCount'.  It's likely that neither a Result Type nor a Result Map was specified.

找不到结果映射类型匹配结果

3.resultMap 与 resultType

mybatis操作中，resultType经常用于存放一个对象，resultMap则是处理结果集(表)中的resultType列名和表对象中的属性名称不匹配。
数据库建表：

CREATE TABLE `t_user` (
  `t_id` bigint(20) NOT NULL AUTO_INCREMENT,
  `t_name` varchar(20) DEFAULT NULL,
  `t_salary` decimal(8,2) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB AUTO_INCREMENT=8 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

@Data
public class User {
    private Long id;
    private String name;
    private BigDecimal salary;
}

此时直接用resultType时会报错的,可以使用以下方法处理。

1) sql语句处理
<select id="listAll" resultType="user">
        SELECT t_id as id,t_name as name,t_salary as salary FROM t_user
</select>
`
2) resultMap ： 结果集合的映射
id: 当前Mapper文件中的resultMap的唯一名称
type: 把结果集中的每一行数据封装成什么类型的对象

子元素: 
    id: 功能和result有一模一样，如果是主键建议使用id元素,提升性能
    result: 匹配对象中的哪一个属性对应表中的哪一个列

    <!-- 配置-->
    <resultMap type="User" id="BaseResultMap">
    <id column="id" property="id"/>
    <result column="name" property="name"/>
    <result column="salary" property="salary"/>
    </resultMap>

    <!--引用-->
    <select id="listAll" resultMap="BaseResultMap">
        SELECT t_id,t_name,t_salary FROM t_user
    </select>

4.查询表中的部分列映射对象使用map

//查询表中一行的所有属性
<select id="get" resultType="hello.User">
    SELECT id,name,salary FROM t_user WHERE id = #{id}
</select>

//查询表中一行的部分属性
<select id="queryForMap" resultType="map">
    SELECT id,name FROM t_user
</select>

5.properties的使用

比较简单的就是放在文件的首部，后面的引用就是直接 ${name}
例如：

<properties>
    <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
    <!--至于这个为什么这么长，是mysql版本的问题-->
    <property name="url" value="jdbc:mysql://localhost:3306/stu?useSSL=false&amp;serverTimezone=GMT%2B8"/>
    <property name="username" value="root"/>
    <property name="password" value="xxxxx"/>
</properties>

properties比较好就是在于可以引用外部文件
例如创建一个db.properties文件,记住不要多空格什么，会出错的。

#key=value
driver=com.mysql.cj.jdbc.Driver
url=jdbc:mysql://localhost:3306/stu?useSSL=false&serverTimezone=GMT%2B8 
username=root
password=xxxxxx

引用该值加入这个(注意路径，我这个引入文件和被引入文件是在一个文件夹下的)

<!-- properties加载外部文件 -->
<properties resource="db.properties"/>

6.Mapper

先上代码:

// 查询t_user表中的结果总数
    @Test
    void testGetCount() throws Exception {
        InputStream in = Resources.getResourceAsStream("mybatis-config.xml");
        // 2.创建Sqlsessionfactory对象。好比是DataSource
        SqlSessionFactory factory = new SqlSessionFactoryBuilder().build(in);
        // 3.创建SqlSession对象，好比是Connection
        SqlSession session = factory.openSession();
       //使用namespace.id的方式去找到SQL元素，并执行SQL语句
        int rows = session.selectOne("hello.UserMapper.queryForCount");
        session.close();
        System.out.println(rows);
    }

    // 查询t_user表中的 id 列和 name 列
    @Test
    void testGetMap() throws Exception {
        InputStream in = Resources.getResourceAsStream("mybatis-config.xml");
        SqlSessionFactory factory = new SqlSessionFactoryBuilder().build(in);
        SqlSession session = factory.openSession();
        List<Map<String, Object>> us = session.selectList("hello.UserMapper.queryForMap", 1);
        for (Map<String, Object> map : us) {
            System.out.println(map);
        }
        session.close();
    }

之前都是使用namespace.id的方法去找到mapper.xml中的SQL元素，并执行SQL语句
这里有几个不足：

  • 传入的参数是String类型的参数，写错的时候只能在运行的时候发现，例如：

    hello.UserMapper.queryForMap
    hello.USerMapper.queryForMap

  • 传入MyBatis需要的参数的类型是不被检查的，因为Sqlsession中传递的参数类型都是Object类型
  • 每一个操作的代码模板是一样的，见上，只有中间的操作语句不同。

解决方案:使用Mapper接口，类似DAO接口，在Mapper接口中去定义每一个操作方法

Mapper组件: Mapper接口 + Mapper文件

Mapper文件和Mapper接口应该放置在同一个包中
Mapper文件中的namespace就设置为对应的Mapper接口的全限定名称
Mapper文件中的操作元素的ID对应Mapper接口中的方法名称

Mapper接口的原理: 动态代理
代理对象: class com.sun.proxy.$Proxy8
具体如何使用动态代理实现的以后再看。

7. 处理mybatis查询传入多个参数

  • 使用JavaBen，就是将参数封装起来成一个对象例如:
    `java
    import lombok.AllArgsConstructor;
    import lombok.Getter;

@AllArgsConstructor
@Getter
public class LoginVO {
private String username;
private String password;
}

Mapper.java文件：
```java
public interface ClientMapper {
    //方式一：把多个参数封装成为javabean
    Client login1(LoginVO vo);
}

Mapper.xml配置文件

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper
  PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="mapper.ClientMapper">
    <select id="login1" resultType="client">
        SELECT id,username,password FROM client 
        WHERE username = #{username} AND password = #{password}
    </select>
</mapper>

具体测试:

@Test
    void testLogin1() throws Exception {
        LoginVO vo = new LoginVO("will", "1111");
        SqlSession session = MyBatisUtil.getSession();
        ClientMapper clientMapper = session.getMapper(ClientMapper.class);
        Client client = clientMapper.login1(vo);
        session.close();
        System.out.println(client);
    }
  • 使用Map对象封装多个参数
    Mapper.java文件：
    public interface ClientMapper {
  //方式二:使用Map对象来封装多个参数
  Client login2(Map<String, Object> paramMap);
}
    Mapper.xml配置
    `java
    <?xml version=”1.0” encoding=”UTF-8”?>
    <!DOCTYPE mapper
    PUBLIC “-//mybatis.org//DTD Mapper 3.0//EN”
    “http://mybatis.org/dtd/mybatis-3-mapper.dtd">



查询测试:
```java
@Test
    void testLogin2() throws Exception {
        Map<String, Object> paramMap = new HashMap<String,Object>() {
            {
                /**
                 * 这里的map中对应的key是与mapper.xml文件的查询语句一样的
                 */
                this.put("username1", "lucy");
                this.put("password1", "2222");

            }
        };
        SqlSession session = MyBatisUtil.getSession();
        ClientMapper clientMapper = session.getMapper(ClientMapper.class);
        Client client = clientMapper.login2(paramMap);
        session.close();
        System.out.println(client);
    }

注意map中的key是与sql语句中查询语句对应的。见图
这样的话，就避免产生了生成过多的javabean。在此基础上，进一步使用注解，就可以减少写map的代码。

  • @Param注解
    Mapper.java
    public interface ClientMapper {
  //方式三:使用Param注解，原理是方式二
  Client login3(@Param("u_name") String username,@Param("u_password")  String password);
}
    Mapper.xml配置
    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper
PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
"http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="mapper.ClientMapper">
  <select id="login3" resultType="client">
      SELECT id,username,password FROM client 
      WHERE username = #{u_name} AND password = #{u_password}
  </select>
</mapper>
    查询测试:
    @Test
  void testLogin3() throws Exception {
      SqlSession session = MyBatisUtil.getSession();
      ClientMapper clientMapper = session.getMapper(ClientMapper.class);
      Client client = clientMapper.login3("will","1111");
      session.close();
      System.out.println(client);
  }
