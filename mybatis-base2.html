1.#和$

Mapper.xml中：
用#和\$获取信息

SELECT id,username,password FROM client WHERE username = #{u_name} AND password = #{u_password}
SELECT id,username,password FROM client WHERE username = \${u_name} AND password = \${u_password}

让我们来看一看后台生成的SQL语句:
使用#:
Preparing: SELECT id,username,password FROM client WHERE username = ? AND password = ?
Parameters: will(String), 1111(String)

使用$:
Preparing: SELECT id,username,password FROM client WHERE username = root AND password = chase
Parameters:

1.1 #和$符号的异同:

相同: 都可以通过 # 和 \$ 来获取对象中的信息。
不同:

  • 使用#传递的参数会先转换为占位符’？’，再通过设置占位符参数的方式来设置值(统统会给值使用单引号引用)
  • 使用\$传递的参数，直接把解析出来的数据作为SQL语句的一部分

$有妙用就是在于稍微复杂的sql语句:
例如sql排序查找:
例如:

SELECT id,username,password FROM client ORDER BY id desc
如果用 # 处理:
SELECT id,username,password FROM client ORDER BY #{orderBy}
假如 orderby 参数传入的是 id desc
执行以后就是 SELECT id,username,password FROM client ORDER BY ‘id desc’ ，该语句无效，
正确的语句应该是:
SELECT id,username,password FROM client ORDER BY \${orderBy}
内部生成的sql语句应该是 SELECT id,username,password FROM client ORDER BY id desc 无引号
所以这里的限定条件是不能用 # 处理的，要用 $

  • 方式一: 使用xml配置，把SQL和映射写在XML文件中 (推荐)
  • 方式二: 使用注解配置，把SQL和映射写在Java代码中(Mapper接口)

这里注意: 关联类的时候路径写’.’，其他文件使用 ‘/‘ 这里开始他一直报错说找不到我的mapper.UserMapper类，就是因为我用了 ‘/‘

UserMapper.java 加注解:

package mapper;

import java.util.List;

import org.apache.ibatis.annotations.Delete;
import org.apache.ibatis.annotations.Insert;
import org.apache.ibatis.annotations.Options;
import org.apache.ibatis.annotations.Select;
import org.apache.ibatis.annotations.Update;

import domain.User;

public interface UserMapper {

    @Insert("INSERT INTO t_user (id,name,salary) values (#{id},#{name},#{salary})")
    @Options(useGeneratedKeys=true,keyProperty="id")
    void save(User u);

    @Update("UPDATE t_user SET salary = #{salary} where id = #{id}")
    void update(User u);

    @Delete("DELETE FROM t_user where id = #{id}")
    void delete(Long id);

    @Select("SELECT id,name,salary FROM t_user WHERE id = #{id}")
    User get(Long id);

    @Select("SELECT id,name,salary FROM t_user")
    List<User> listAll();
}

2.1 注解中如何使用ResultMap

xml文件中是有resultmap来处理表名与我们创建的实体类名不匹配的问题，那么注解的实现
假设数据库创建语句:

CREATE TABLE `t_user` (
  `u_id` bigint(20) NOT NULL AUTO_INCREMENT,
  `u_name` varchar(20) DEFAULT NULL,
  `u_salary` decimal(8,2) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB AUTO_INCREMENT=12 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

@Select("SELECT u_id, u_name, u_salary FROM t_user WHERE u_id = #{id}")
@Results(id = "BaseResultMap", value = {
            @Result(column="u_id", property="id"),
            @Result(column="u_name", property="name"),
            @Result(column="u_salary", property="salary")

    })
User get(Long id);

那么下一次引用就不用写这么一串了，不用重复写:

@Select("SELECT id,name,salary FROM t_user")
@ResultMap("BaseResultMap")
List<User> listAll();

@Select()注解中，如果语句过长，是可以分开写的，因为内部是String数组，例:

@Select({"SELECT u_id, u_name, u_salary FROM t_user"," WHERE u_id = #{id}"})

实际开发中xml更多，而不是注解

3.1 if的使用

目的：查询工资大于等于900 小于 1200的员工

//这里使用@Param传参就很舒服
public interface EmployeeMapper {
    List<Employee> query2(
            @Param("minSalary") BigDecimal minSalary,
            @Param("maxSalary") BigDecimal maxSalary
            );
}

Mapper.xml配置(不使用注解):

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper
  PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="mapper.EmployeeMapper">
    <select id="query2" resultType="Employee">
        SELECT id, name, sn,salary,deptId FROM employee
        <if test="minSalary!=null">
            WHERE salary >= #{minSalary}
        </if>
        <if test="minSalary!=null">
            AND salary &lt;= #{maxSalary}
        </if>
    </select>
</mapper>

这里有一个坑点就是 ‘<’符号放在sql会被读成 标签开始的 符号需要用 &lt表示’<’，见上语句

测试语句:

// 查询工资大于等于900 小于 1200的员工
@Test
    void test2() {
        SqlSession session = MyBatisUtil.getSession();
        EmployeeMapper em = session.getMapper(EmployeeMapper.class);
        List<Employee> ems = em.query2(new BigDecimal(900),new BigDecimal(1200));
        for (Employee employee : ems) {
            System.out.println(employee);
        }
        session.close();
    }

3.2 choose的使用

Mapper接口:

public interface EmployeeMapper {
    List<Employee> query3(
            @Param("minSalary") BigDecimal minSalary,
            @Param("maxSalary") BigDecimal maxSalary,
            @Param("deptId") Long deptId
            );
}

Mapper.xml

<select id="query3" resultType="Employee">
        SELECT id, name, sn,salary,deptId FROM employee
        WHERE 1=1
        <if test="minSalary!=null">
            AND salary >= #{minSalary}
        </if>
        <if test="minSalary!=null">
            AND salary &lt;= #{maxSalary}
        </if>
        <choose>
            <when test="deptId > 0">AND deptId = #{deptId}</when>
            <otherwise>AND deptId IS NOT NULL</otherwise>
        </choose>
    </select>

注意这里WHERE 1 = 1的使用，感觉是很精巧了,但是很low

3.3 where元素的使用

作用: 判断查询条件是否有WHERE关键字，若没有，则在第一个查询之前插入一个WHERE关键字;如果发现查询条件以AND或者OR开头，也会把第一个查询条件前的AND/OR替换成WHERE
所以说上面的WHERE 1 = 1就用where代替了

前提：trim元素包含内容返回了一个字符串 * prefix: 在这个字符串之前插入了一个prefix属性值 * prefixOverrides: 如果字符串的内容以prefixOverrides中的内容开头(可以包含管道符号)，那么使用perfix属性值替换内容的开头 * suffix: 在这个字符串之后插入suffix属性值 * suffixOverrides: 如果字符串的内容以prefixOverrides中的内容结尾(可以包含管道符号)，那么使用suffix属性值替换内容的结尾 可以代替set和where，但是不会这么做xml
